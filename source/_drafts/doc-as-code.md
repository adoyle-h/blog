## 概览(Overview)/前言(Intro)

无论什么文档，只要规模上涨，维护就会变得非常麻烦。

<!-- more -->


我觉得 Literate programming 实际上仅仅是把文档和代码写在同一个文件。
而我认为理想的“代码即文档”是用代码来表述文档，即文档是根据代码自动产生的，而代码本身也可以完成自身业务逻辑。
所以代码一旦变化，文档也会随之改变。这样我们只需要改动一个地方就行了。
